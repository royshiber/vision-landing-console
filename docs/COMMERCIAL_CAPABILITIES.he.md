# Vision Landing Console — מאגר יכולות וערך עסקי

מסמך מקור **לבעלי עניין מחוץ לצוות הפיתוח** (שותפים, קהילה, מהנדסים, וגם מי שאינם מהנדסים — הרעיון הוא לבנות **את אותה הפשטה** ש־Cursor הביא לעולם הפיתוח).  
**הטון והחזון הרחבים:** לעולם הכטב"מים, הרובוטיקה והמערכות הבלתי־מאוישות — **מה ש־Cursor עשה לעולם קוד**: סביבת עבודה אחת חכמה ומחוברת, מרעיון ועד החומרה, שמאפשרת למהנדסים **ולא־מהנדסים** לבנות ולהפעיל בהירות ובטיחות. **נאמר במפורש:** אנחנו **עדיין לא** בשלב “הכל מושלם כמו ההבטחה” — מה שיש בהמשך המסמך הוא מה שכבר רץ ובנוי; הצפון מספק את **הפריים** למסחר ולקבוצה.

- **גרסת מוצר (מקור אמת למספר גרסה):** ראה `version.js` / תג גרסה ב־UI.  
- **עדכון מסמך:** כאשר מתווספת יכולת משמעותית למוצר או מתחלפת אסטרטגיה — לעדכן את הקטעים הרלוונטיים בסעיפים בהמשך וב־`public/changelog.json` כמקור משלים להיסטוריית שחרורים.

## חזון: Cursor לכטבמים ורובוטיקה (צפון מוצר)

**למה בעצם נלחמים בסיפור העסקי:** לעולם פיתוח **כלי הטיס**, הרובוטיקה והמערכות האוטונומיות — **את אותו סוג מהפכה** שכלי כמו Cursor נתנו לפיתוח תוכנה: לא רשימה של חלון־כלים נפרדת, אלא **סביבה אחת** שמרכזת הקשר טכני (קוד וחומרה), מקצרת מחזורים, והופכת מתכון מורכב למשהו שאפשר **לגלוש בו בשפה אנושית**.

- **מהנדסים** — עומק, שליטה, audit, פרמטרים, חיבורים לג’טסון ו־MAVLink.  
- **לא־מהנדסים (במרכז המשקל העסקי)** — הפחתת רף דרך **שיחה, קול, הסברים, אשף קונפיגורציה** — עם גדר בטיחות בשרת (Apply מבוקר — לא הבטחה של “כל מילה נכנסת לבקר”).

**מה יש בתוך הפרויקט היום לאור החזון:** קונסולה שמממשת שכבות ליבה (ArduPilot, companion, פרמטרים, יועץ, זיכרון, מהנדס קולי, ArduLab) — **בסיס** נטו לצמיחה אל “הסביבה המלאה”; כפי שכל גרסה של Cursor הציבה הרבה בצעדי ביניים, כך תיאור הגירסאות בהמשך מניע את הסיפור.

## 1. מטרת המסמך

לאפשר דיון עסקי ומוצרי **בלי צורך לקרוא אלפי שורות קוד**: מה המערכת עושה, למי זה מתאים, מה נשמר והיכן רואים שהשינוי אכן התבצע, ומה המגבלות.

---

## 2. מהות המוצר במשפט

**שרת ניהול ארוג מקומי (Node.js) עם ממשק דפדפן**, המרכז עבודה עם **ArduPilot** בסביבת **נחיתה ויזואלית** וחיבור **Jetson/companion**, ושכבות **יעוץ וביצוע פרמטרים מבוקר** (Gemini והקשר טיסות/לוגים), **שיחה קולית** (מהנדס טיסה), יצירת **פיצ’רים מותאמים** עם קוד פרמטרים, ובסיס SQLite לזיכרון ו־audit.

---

## 3. קהלי יעד והבטחת ערך

| קהל | צורך | איך המוצר מתיישב |
|-----|------|-------------------|
| **חובבי FPV ומערכות מתקדמות** | ניסוי מהיר של פרופיל פחות מתסריט אחיד | טאבי פרמטרים, HUD, משוב AI בלי כניסת מומחה |
| **חברות/צוותים סביב ArduPilot** | שקיפות שינויי פרמטר, הדרכה, לוגים | יועץ עם audit, Jetson API, מסמכי בטיחות |
| **מפעילי שדה** | קוקפיט שמדבר עברית/אנגלית, פחות כפתורים | מהנדס טיסה קולי, פנקס הערות |
| **מפתחים / יזמים טכנולוגיים (“לא מהנדסי קושחה מלא”)** | לתרגם רעיונות לשכבת ArduPilot בלי שבועות של כוונון ידני | ArduLab, יועץ, קונפיג אוטומטי, ובטיחות שרת שמונעת “ניסוי עיוור” |

## 4. מבנה ארכיטקטוני בסיס

- **לקוח:** SPA סטטי תחת `public/` (HTML/CSS/JS), נטען מ־Express.
- **שרת:** `server.js` + רישום מרוכז ב־`lib/routes/http-register.mjs` (ליבה, יועץ, Feature Designer, מהנדס טיסה).
- **מסד:** SQLite (`data/vision-landing.sqlite` כברירת מחדל) — טיסות, לוגים, שיחות יועץ, audit פרמטרים, פיצ’רים מותאמים, זיכרון מהנדס.
- **חומרה:** חיבורי סריאל/Mavlink דרך שכבת `mavlink-connection`, מצב לוויין בזיכרון מעודכן מ־companion.

## 5. משטח המוצר — טאבים ותתי־משימות

### 5.1 מרכז פרמטרים (`#control`)

**מטרה עסקית:** מקום אחד לכוונון נחיתה, בטיחות, ניווט ויזואלי, ופרמטרי ArduPilot — עם סנכרון **לשרת** (פרופיל נשמר) ו**לרחפן** (READ/WRITE MAVLink).

- תתי־אזורים: נחיתה, בטיחות/ABORT, ניווט לפי תמונה, **ArduPilot** (קטגוריות EKF, Jetson, PLND, וכו’), **Custom שלי** (פרמטרים מפיצ’רי ArduLab), **קונפיגורציה אוטומטית** (אבחון AI מונחה רכיב).
- כלים: ייצוא/ייבוא JSON, הורדת `.param`, חיפוש טקסט, **חיפוש חכם** (Gemini), השוואה לרחפן אחרי READ.

### 5.2 תחקור (`#recordings`)

הקלטות וידאו, אנוטציות, קישור לטיסות/לוגים — לניתוח אחרי האירוע.

### 5.3 טלמטריה (`#telemetry`)

לוחות סטטוס, בדיקות מערכת, לשוניות משנה (למשל Jetson) — נראות תפעולית.

### 5.4 AI Advisor (`#advisor`)

צ’אט עם רשימת נושאים (issues), הקשר טיסה, הצעות פעולה מובנות, Apply/Rollback, קישור ל־Audit. עיצוב v2 עם סיידבר שמאלי (LTR shell) לעקביות.

### 5.5 ArduLab (`#featureDesigner`)

מעבדה לתיאור פיצ’ר בשפה חופשית, יצירת קוד C++ מוצע, פרמטרים מותאמים, הפעלה/השבתה לחיפוש החכם.

### 5.6 מהנדס טיסה (`#flightEngineer`)

שיחה קולית בזמן אמת, TTS (ברירת מחדל ElevenLabs כשמוגדר), פנקס פתקים לסשן, Apply פרמטרים תחת אותן שכבות בטיחות — **וגם התייעצות על הפרויקט והפרמטרים בלי חיבור לבקר** (שולחן עבודה / הכנה לטיסה). זיהוי דיבור: עברית / אנגלית / מנדרינית (`zh-CN` או ניחוש שפה לפי תוכן הדיבור); התנהגות המודל מוטנית להקשר טיסוני בעברית, ובסינית אופציה למענה עם **פיןְיין** ליד מונחים טכניים.

### 5.7 מעבדת SITL (`#simLab`)

חיבור MAVLink ל־**ArduPilot SITL** (תהליך חיצוני — לינוקס/WSL) עם קיצורי UDP/TCP, תצוגת מטוס תלת־ממדי (**פרימיטיב או GLB מהשרת**; **בדיקת גודל לפני טעינת GLB** ואזהרה לא חוסמת לקבצים גדולים), מפת ריפליי עם **OSM / לוויין / גרדיאנט מקומי**, **פרופיל מסומן** ArduPlane + Jetson bundle (מקטלוג שחרורים), **כתיבת פרמטר חיה** דרך `param-set` כש־MAVLink מחובר (שכבות ARM כמו בייצור), **שיחזור ‎.tlog** עם **אירועי ציר מהלוג** (STATUSTEXT, ARM/DISARM, שינויי ‎flight_mode‎ עם **שמות מצב ArduPlane**), ואופציה ל־**MANUAL_CONTROL** מהמשחקון דרך השרת (`ALLOW_BROWSER_MANUAL_CONTROL`) — ל־SITL בלבד. **תאימות import maps** (Three): טעינת **es-module-shims** לפני מפת הייבוא. **HUD חי** משלים גובה/מהירות/כיוון מ־GLOBAL ו־ATTITUDE כש־`VFR_HUD` דליל — עם סימון **SIM** כשמהירות היא פרוקסי מ-groundspeed (לא IAS מפיטו), ובקרת התאמת זמן חבילות (**skew**) בשידור SSE. ראה `docs/SITL_AND_SIM_LAB.md`.

**Photo→Mesh (P3):** קיים נתיב `POST /api/sim-lab/mesh-job` לאימות נכס תמונה וסביבת ספק; השלמת קריאות SaaS אינה מוצגת כ«סופק במלואו» עד יישום צד השרת.

#### צפון קצר — נכס תלת־מימדי למטוס

- **P0–P2:** ממומש בממשק וב־API (העלאה, GLB, לוויין/גרדיאנט).
- **P3:** בסיס שרת + מפתחות `.env`; אינטגרציית ספק מלאה — צפונה.
- **חיזוק נכסים:** אופטימיזציית glTF לפני טעינה — מתוכנן.

### 5.8 הטסה (`#terrain`)

מפה, HUD, פקודות טיסה — ניסוי ויזואלי בשטח (תלוי תצורת רשת/מפה). שדות נתונים מותאמים ב-HUD: זיהוי מנוחש (מקומי + Gemini) עם **בירור ובחירה** כשהביטוי דו-משמעי. **הודעות FC (STATUSTEXT)** מוצגות בעברית ברורה (תרגום Gemini); **כפתור קול** פותח את מהנדס הטיסה; **חיווי ARM** לחיץ להסבר מוכנות.

### 5.9 לוגים והעלאות (`#flights`) — מוסתר מה־nav

נגיש programmatically; ניהול טיסות, העלאת לוגים, קישור ליועץ.

### 5.10 תהליכים (`#processes`)

פאנל תהליכים/מצב (תלוי יישום UI) — ניטור פנימי.

## 6. יכולות בינה והקשר ארוך־זמן

| מסלול | תיאור עסקי |
|--------|------------|
| **יועץ טקסט** | שאלות על נדנוד, נחיתה, Jetson, פרמטרים; retrieval מלוגים/מסמכים לפי תצורה |
| **חיפוש חכם בפרמטרים** | שאילתה חופשית → דירוג פרמטרים רלוונטיים (מנועי smart search / embeddings לפי גרסה) |
| **קונפיגורציה אוטומטית** | תיאור תקלה לפי רכיב → תוכנית בדיקות והמלצות פרמטר עם Apply מבוקר |
| **ArduLab** | שלב מחקר/פיתוח לפיצ’ר מותאם עם ניהול חיים (טיוטה/הפעלה) |
| **מהנדס טיסה** | דיאלוג קולי + אותו שכבת retrieval מהיועץ (docs, מסד פרמטרים, audit, digest — מכווץ) + debrief לסשן הבא |

**הגבול המסחרי:** שירותי Gemini/ElevenLabs הם **עלות משתנה**; יש **throttle** ליועץ (`express-rate-limit` על `/api/advisor-chat`) ו־**מינימום זמן בין קריאות** אופציונלי ב־`.env` לשליטה בעלות.

## 7. Jetson companion, חזון, ו MAVLink

- **דופק חיים:** `POST /api/jetson/heartbeat` (ואחורה `/api/rpi/*`) — חיבור סטטוס Jetson ללוח הבקרה.
- **תמונות ומצב SLAM/VIO:** נקודות קצה ל־frame, pose, optical flow — לניווט ללא/עם GPS חלש (פירוט הנדסי ב־`docs/JETSON_AGENT.md`).
- **MAVLink:** ניהול חיבורים, בקשת פרמטרים, כתיבה מבוקרת, משימות; ערוץ אישור חומרה אופציונלי לפרמטרים דרך מהנדס (`FE_APPROVAL_RC_CHANNEL`).
- **עסקית:** חברה שמוכרת ערכת Jetson יכולה להציג את הקונסולה כ־**ממשק אחיד** מול הלקוח יחד עם חומרה.

## 8. מרכז פרמטרים, חיפוש חכם, וקונפיגורציה אוטומטית

- **סכימה וגבולות:** `param-schema` ו־allowlists — עוגן לשינויים בטוחים.
- **חיפוש חכם:** API `/api/param-center/smart-search` — מנועים `param-smart-search` / v2, semantic לפי embeddings כשבנויים.
- **קונפיגורציה אוטומטית:** `auto-config-recipes` + תכנון AI — מפות חומרה מובנות לתרחישים.

## 11. מהנדס טיסה (קול)

- API: צ’אט, TTS, פתקים per session, debrief, Apply פרמטר, סטטוס.
- **הגדרות כלליות** (כפתור גלגל בשורת הכותרת): נפח שמע, בחירת קול ElevenLabs (רשימה מהשרת כשמוגדר מפתח), קול דפדפן כגיבוי, דילוג על שפת זיהוי דיבור — עם תזכורת להרשאות עריכה (יועץ / מהנדס) ומצב כתיבה בזמן טיסה לפי השרת.
- **בלי חיבור לבקר / בלי טיסן באוויר:** ייעוץ על ארכיטקטורת הקונסולה, משמעות פרמטרים, הכנה לטיסה והדרכת הפעלה — עם אותו שכבת retrieval וזיכרון; נתונים חיים מהמטוס רק כש־MAVLink מחובר ופרמטרים נטענו.
- **הקשר כמו היועץ (RAG):** באותו סשן נשלחים למודל גם מסמכי `docs/`, חיפוב מסד פרמטרים ArduPlane, audit שינויים אחרונים, פרופיל Jetson בשרת, קטעים ממאגר טיסות/לוגים (אופציונלי לפי `flightId`), ו-digest מ-GitHub — בדומה ליועץ הטקסטואלי, בנפח מכווץ לקול.
- **זיכרון ארוך־טווח:** פרופיל מהנדס, אירועים, debrief — בטבלאות ייעודיות.
- **TTS:** ElevenLabs (עברית ושפות נוספות) או נפילה לדפדפן — לפי מפתחות ב־`.env`.

## 12. זיכרון, audit, ובטיחות Apply

- **יועץ:** אין כתיבה ישירה לבקר מהדפדפן — רק **Apply** מאומת בשרת; `param_audit` append-only; snapshots ל־rollback.

## 21. מאגר נתונים — ישויות עסקיות

המסכים נשענים על SQLite. להלן **מה נשמר ולמה זה חשוב לשיחה עסקית** (לא סכימה מלאה של עמודות).

| ישות | ערך למשתמש/עסקים |
|-------|-------------------|
| `flights`, `flight_notes`, `log_artifacts` | ניהול טיסה, הערות, והקשר לוגים לצורך ניתוח ויועץ |
| `code_digest` | מנגנון של GitHub ingest — “מה השתנה בקומיט” לרמות עדכון ידע יועץ |
| `chat_issues`, `chat_messages`, `chat_actions` | הנפקות שיחות, פתרון נושאים, כרטיסיות פעולה לפני Apply |
| `param_audit`, `param_snapshots` | עקיבות פרמטרים; rollback מוכח |
| `session_baselines` | בסיס הפעלה ל-session — הצגת שינויי ממתין למדיניות revert |
| `jetson_profile` | פרופיל פרמטרים ציד Jetson מתועד במקום להישאר רק בדפדפן |
| `server_config`, `terrain_cells` | הגדרות שרדות ומרשת טריין (יכול לעזור בשחזור מפות) |
| `custom_features`, `custom_params` | יצירות ArduLab — קוד, פרמטרים, הפעלה |
| `engineer_notes`, `engineer_profile`, `engineer_events`, `engineer_session_debriefs` | מהנדס קולי — זיכרון ארוך־טווח וסיכומי אחרי טיסה |
| `aircraft_model_assets` | מטא־דאטה לקבצי מטוס (צילום / GLB) שנשמרו תחת `data/uploads/` — בסיס לצינור תלת־מימד במעבדת SITL |
