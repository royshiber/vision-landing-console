# מעבדת SITL / סימולציה — תיעוד ייחוס טכני

מסמך מרכזי לסביבת בדיקות **SITL** מול Vision Landing Console: מטרה, מה ממומש, תוכנית, מפת UI, API וקבצים. תקציר מוצרי מקוצר: [`docs/COMMERCIAL_CAPABILITIES.he.md`](COMMERCIAL_CAPABILITIES.he.md) — **§5.7** (`#simLab`).

---

## מטרה

| נושא | תיאור |
|------|--------|
| **לוגיקת טיסה** | התנהגות זהה לשדה מגיעה מ־**קושחת ArduPilot אמיתית** ב־**SITL**; המסוף הוא GCS אחרי **MAVLink** — כמו למטוס חי. |
| **ויזואליזציה ב‑VLC** | התלת־ממד בדפדפן הוא **תצוגה מהירה מהטלמטריה** (ועליה בזמן **ריפליי**), בלי להחליף את הפיזיקה של ArduPilot. |
| **לא תחליף לסימולטור כבד** | רמת RealFlight / **Gazebo / FlightGear / X‑Plane** — זוגות עם SITL וערוצי משוב חיצוניים, מחוץ ל־VLC. |

## מה קיים במוצר

| יכולת | פרטים |
|--------|--------|
| **חיבור MAVLink** | טאב «מעבדת SITL»: קיצורי **UDP/TCP** (למשל `127.0.0.1:14550`, `127.0.0.1:5760`); החיבור הפעיל נשאר בווידג'ט CONNECT הראשי של המסוף. |
| **פרסטי «מחסנית» (תיעוד בלבד)** | `GET /api/sim-lab/stack-presets` — קו **ArduPlane** מסומן במעבדה + גרסאות **Jetson bundle** מ־`lib/jetson-releases.mjs` (HUD/ציפיות, לא החלפת בינארי SITL אצלך). |
| **תלת־ממד + מצב «סימולטור»** | שמיים/ערפל/מסלול, מצלמת מעקב; מצב ויזואלי מועשר לעומת מצב קלאסי (toggle בממשק). **ברירת מחדל:** פרימיטיב גאומטרי; **אופציונלי:** טעינת **GLB** מהשרת (`/uploads/…`) עם קנה־מידה אוטומטי (ראו טאב «מודל מטוס תלת־ממד»). |
| **פרמטר חי בזמן SITL** | טופס במעבדה → `POST /api/param-center/param-set` → **PARAM_SET** בקושחה (כמו בייצור). **ARMED** או ARM לא ידוע → **409** (אותה שכבת בטיחות כמו בשאר ה‑UI). |
| **ריפליי מ־`.tlog`** | העלאה → `POST /api/sim-lab/parse-tlog` (עד ~**85 MiB** בזיכרון); דגימות מצומצמות (ATTITUDE / גובה / מהירות מ־VFR או GLOBAL / GPS כשיש) + **`replayEvents`**: STATUSTEXT, ARM/DISARM, נקודות שינוי **flight_mode** מ־heartbeat עם **שם מצב ArduPlane** (מפת `custom_mode` המסונכרנת עם ה־HUD) (**עד 400** אירועים לתצוגה). בצד השרת: עד ~**22k** דגימות, **דגימה ≥70 ms** בין נקודות לאורך ציר הזמן. |
| **מפה** | אם יש GPS בדגימות — Leaflet במעבדה; **שכבות בסיס**: OSM / לוויין Esri / גרדיאנט מקומי (בלי מפתחות). מסלול מסונכרן עם **טאב «הטסה»**. |
| **MANUAL_CONTROL מהדפדפן** | כבוי כברירת מחדל. **למעבדת SITL בלבד:** `ALLOW_BROWSER_MANUAL_CONTROL=1`, סימון «שליחה לקושחה», משחקון USB ו/או **WASD + Q/E + [ ] לדחף** (המקשים מפורטים בממשק). שאילתת יכולת: `GET /api/sim-lab/rc-capability`. שליחה: `POST /api/mavlink/manual-control`. **לא לטיסה אמיתית.** |
| **נכסי מטוס (GLB / צילום)** | `POST /api/aircraft-model/upload` + `GET /api/sim-lab/aircraft-models` — רשימת נכסים והורדה דרך `/uploads/`. בממשק: העלאה, בחירה, החלה על סצנת Three.js — **גלילת הנכסים מאגדת רק ‎GLB‎ להחלה בכפתור**; העלאת **צילום (JPEG וכו׳)** מחילה **מרקם על הפרימיטיב** בלבד (לא יוצאים מפת mesh תלת־ממדית מתוך תמונה בלבד בסוכן) **לפני טעינת GLB:** אימות גודל (עד ~**45MB**; אזהרה לא חוסמת מעל **8MB**; מקור: `sizeBytes` מהשרת או `HEAD` ל־`/uploads/…`). תגובת העלאה: `downloadUrl` ו‑**`url`** (ערך משוכפל לממשקות). |
| **צינור mesh (P3)** | `POST /api/sim-lab/mesh-job` — בודק נכס תמונה + מפתחות ספק ב־`.env`; **501** עד יישום קריאות SaaS. `GET /api/sim-lab/capabilities` — דגלים תצורתיים. |

**ריפליי — דיוק ציפייה:** נתמך היטב לקבצי **`.tlog`** טיפוסיים (MAVLink + אופציונלית חותמת Mission Planner לפני החבילה). סיומות אחרות בוחרות ב‑UI — השרת מנסה לפרסר MAVLink מהבאפר; **DataFlash `.bin`** הוא פורמט אחר — לריפליי ייעודי במעבדה עדיין לא.

**HUD חי (SSE):** כש־`VFR_HUD` דליל או חסר (קישור USB איטי וכו'), גובה / מהירות / כיוון משלימים מ־**GLOBAL_POSITION_INT** ומ־**ATTITUDE** באותה לוגיקה כמו ב־`lib/mavlink-hud-fields.mjs` (גם ליועץ ולמהנדס טיסה). כשאין `airspeed` מ-VFR ומהירות המוצגת היא **פרוקסי מ-groundspeed**, הממשק מסמן **SIM** (צבע/כיתוב) — לא מתאים להסקות סטול IAS אמיתי ב-ArduPlane בתנאי רוח. כשפער זמן הקבלה בין זרמי MAVLink גדול, נדלק **`hudTimeSkewWarn`** ומתכנס ערבול הגיאומטריה מול הגלובלי כדי להפחית «קריעת» HUD בין אופק לגובה.

| נושא | פשרה |
|------|--------|
| **IAS ללא פיטו** | חוסר `airspeed` מ־VFR → משלימים מ־**groundspeed** (מוגבל פיזיקלית לעומת IAS אמיתי). ב-UI מסומן **SIM**; אין להסיק סטול / מרווחי בטיחות כמו מול פיטו. |
| **סינכרון HUD בין חבילות** | זמני קבלה ו־`time_boot_ms` שונים מדי בין ATTITUDE לבין GLOBAL/VFR — ערבול צמתים מצומצם ודגל **`hudTimeSkewWarn`** ב-SSE. |
| **גובה** | עדיפות: VFR alt → יחסי GLOBAL → MSL GLOBAL. |
| **כיוון** | VFR heading → GLOBAL hdg → yaw מ־ATTITUDE (0…360°). |
| **Three.js** | `sim-lab.mjs` כמודול + **import map** לנתיבים תחת **`/vendor/…`** (קבצים מועתקים מ־`node_modules` באמצעות **`npm run vendor:sync`** — ללא curl/wget). לפני כן נטען **es-module-shims** מאותו נתיב — מתאים ל‑**air‑gap** אחרי סנכרון vendor מכונה מחוברת. |
| **טעינת GLB** | רק נתיבי `/uploads/…` מהאותו מקור (אין כתובות חיצוניות שרירותיות). |
| **לוויין** | Esri World Imagery ציבורי — עומס ותנאי שימוש באחריות המפעיל. |
| **VFR חלקי** | `parseVfrHud` שומר שדות תקינים גם כששדה float אחר לא סופי — להימנע מאיבוי מלא של HUD. |

**ציפיות פוטוגרמטריה / AI:** אין «קסם חינם» — דיוק גבוה דורש צילום רב־זוויתי + כלי מקצועי או SaaS בתשלום; ראו דיון מוצרי ב־§5.7 ובמסמך המסחרי.

## לולאת Vision סגורה (Closed-loop Vision)

| היבט | תיאור |
|------|--------|
| **חזון** | סטרים וידאו **סינתטי** (רינדור בדפדפן וב־או בשרת) מוזרק לצינור ה־Vision ב־**Jetson**; אלגוריתמי הנחיתה פולטים פקודות שחוזרות ל־**SITL** כמשוב על כלי — לולאה סגורה לאימון והדגמה. |
| **קיים כיום (חד־כיווני)** | המסוף מקבל **מידע מה־MAVLink** (מצב, פוזה, מצבי טיסה) מה־FC **אל** ה־UI (כולל **ריפליי** מתוך ‎`.tlog`‎); אין **שידור** וידאו סינתטי אל פייפליין Vision מקומי/מרוחק, ואין סגירת מעגל של פקודות מה־Vision **חזרה** אל פיזיקת SITL כחלק אינטגרלי של הצינור. |
| **דורשים לאינטגרציה** | **סנכרון פריים↔פוזה** (ציר זמן אחיד לרינדור מול MAVLink ATTITUDE / מיקום גלובלי); ניהול **חביון** בכל שלבי הרינדור, הרשת והמשוב; **גשר RTP / WebRTC** (או שכבת מקור מתאימה) מהמוצא הוויזואלי של הקונסולה אל הצריכה ב־**Jetson**, כמו **תַחְלִיף למצלמת USB**. |

להמשך תקציר מוצרי של המעבדה והבטחות ללקוח: **§5.7** במסמך [`docs/COMMERCIAL_CAPABILITIES.he.md`](COMMERCIAL_CAPABILITIES.he.md).

## צינור נכסים אוטומטי — חיזוק P0

העלאת **GLB/GLTF** כבדים בשילוב **ריפליי ב־WebGL** טוענת זיכרון ו־GPU; נכס לא ממוּטב עלול להביא ל**קריסת דפדפן**. בשלב חיזוק **P0**: קו `--fix` בסגנון **gltf-pipeline**, אופטימיזציה ב־**Meshoptimizer** וקומפרסורים סטנדרטיים ל־glTF 2, אופציונלי **Draco**, יחידות **LOD** וצמצום טופולוגי (**decimation**) מתוכננים, עם ספי גודל/זיכרון לפני שחרור ל־UI — יציבות לפני דיוק הנדסי של המודל.

## Fault injection (תוכנית יכולת)

יכולות כמו **`COMMAND_LONG`**, הזרקת תקלה בחיישנים (**GPS drop**, הפרעות קישור), תרחישי דימוי בסגנון **`SIM_ENGINE_FAIL`** או שינויי פרמטר מבוקרים — ראויות לתוכנית המעבדה כלי **בדיקת עמידות** בלבד ל־SITL/HIL. **אין למכור אותן כיכולות חיות ב־UI** כל עוד לא ממומשו בשרת ובממשק בצורה מפורשת ובטיחותית — בקוד הנוכחי **אין** מסלול מובנה בקבוצת **‎`/api/sim-lab/*`** להזרקות כאלה.

## מפת דרך מתקדמת (P4–P6)

| שלב | כיוון |
|-----|--------|
| **P4** | זירות נחיתה מבוססות **NeRF / Gaussian splatting**: סינתזה סביבתית ארוכת־זמן לבדיקות חזותיות חוזרות. |
| **P5** | **HIL ב־WebRTC**: צינור וידאו + פוזה **Jetson↔SITL** להרצת לולאות סגורות מול קושחת ArduPilot בלי שטח. |
| **P6** | **CFD עד משטח ארודינמי / mesh**: מתווה התממשקות **Mesh→מקור כוח ברוח** אל שכבות פיזיקה שמזינות בהמשך (**ArduPilot / SITL**) — טעון מאמץ הנדסי וולידציה מול מתמטיקת הקושחה. |

## החסם העיקרי ל־**P5** (לולאת Vision סגורה)

**P5** נבלם בעיקר ב־**חביון ובלתי־דטרמיניזם** של הרכיבים הדפדפניים ושל גשרי **WebRTC** מול לולאות בקרה מהירות; בדרישה ל**צינור תצוגה–פוזה MAVLink מהימן לכל פריים** (מתויגים לפי בסיס זמן לווידוא); ובאינטגרציה של **גשר סטרמה על ג’טסון** (למשל GStreamer + צומת `appsrc`/קליטה שמחקה צינור **מצלמת USB**) הפועל בשפה ובתזמון אחידים לשאר המערכת. מהצד ההנדסי נדרש גם **עומס אימות מערכתי**: רגרסיות רשת, עומסי CPU/GPU ו**סשני משחזור/אימון** ארוכים — לפני הצגת לולאה סגורה כיכולת שיווק מאושרת.

## מפת דף מעבדת SITL (`#simLab`)

| אזור | תוכן |
|------|------|
| **כותרת / היכרות** | קישור לתיעוד זה; הסבר קצר: פיזיקה ב‑SITL, לא ב‑WebGL. |
| **מסילה שמאלית** | קיצורי חיבור UDP/TCP; בחירת **ArduPlane** + **Jetson**; **פרמטר חי**; **מודל GLB** (רשימה, העלאה, החלה על הסצנה). |
| **מרכז** | **Canvas** תלת־ממדי (פרימיטיב או GLB); HUD; טוגל ויזואליזיה; **מפת ריפליי** עם בחירת רקע (OSM / לוויין / גרדיאנט). |
| **מסילה ימנית** | **MANUAL_CONTROL**: toggle שליחה, רמזים, סטטוס משחקון, מקרא מקשים; **העלאת לוג**, ניתוח, ניגון/השהיה/קצב, פס אירועי `replayEvents`. |

החיבור ל‑**FC הראשי** נשאר בפאנל העליון של האפליקציה — המעבדה רק מקלה על הזנת כתובת/פרוטוקול.

## API חשובים

| שיטה ונתיב | תפקיד |
|------------|--------|
| `GET /api/sim-lab/stack-presets` | פרסטי ArduPlane + רשימת Jetson מהקטלוג. |
| `GET /api/sim-lab/rc-capability` | `{ manualControlAllowed }` לפי env. |
| `GET /api/sim-lab/aircraft-models` | רשימת נכסים מ־SQLite (עם `url` תחת `/uploads/`). |
| `GET /api/sim-lab/capabilities` | `{ aircraftUpload, photoMeshEnvConfigured }`. |
| `POST /api/sim-lab/parse-tlog` | מעלה קובץ (שדה `file`); מחזיר `samples`, `replayEvents`, `durationMs`, `frameCount`. |
| `POST /api/sim-lab/mesh-job` | גוף: `{ assetId }` — נכס מסוג `photo`; **501** כל עוד צד הספק לא ממומש (דורש מפתחות ב־`.env`). |
| `POST /api/mavlink/manual-control` | גוף: `x,y,z,r,buttons` — דורש `ALLOW_BROWSER_MANUAL_CONTROL=1` וחיבור פעיל. |
| `POST /api/aircraft-model/upload` | שדה `file`; אופציונלי `notes` — דורש הקשר DB+upload; 415 לסוג לא נתמך; בחזרה גם `sizeBytes` לאימות גודל בצד הדפדפן. |
| `POST /api/param-center/param-set` | כתיבת פרמטר חיה (שכבות ARM — כמו בשאר המערכת). |

## הרצת SITL (לינוקס / WSL) והתחברות

1. ArduPilot לפי התיעוד הרשמי (למשל `sim_vehicle.py`).
2. דוגמה (התאם נתיב ופרוטוקול):  
   `sim_vehicle.py -v ArduPlane --console --map --out udp:127.0.0.1:14550`  
   (חלק מההגדרות משתמשות ב־**TCP** במקום UDP — התאם לפורט שלך.)
3. ב־Vision Landing Console: טאב **מעבדת SITL** → קיצור **UDP 127.0.0.1:14550** (או TCP מתאים) → **Connect** בפאנל העליון.

### שלט בזמן SITL

- **יציב:** MAVProxy / `sim_vehicle` עם מפת `joystick` — USB ישירות ל‑SITL.
- **VLC:** כשהדגל מופעל והרישוי מאושר, `MANUAL_CONTROL` דרך הדפדפן והשרת — לוודא מצב טיסה ופרמטרים (למשל `MANUAL`).
